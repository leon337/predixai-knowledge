# GIT_SITE_SYNC_POLICY

## Regra oficial
A partir da V2.3, toda etapa publicada no GitHub deve atualizar também o site em `docs/`.

## Fluxo obrigatório
1. Alterar código/documentação.
2. Atualizar Markdown oficial.
3. Atualizar página HTML correspondente.
4. Validar links.
5. Validar diff.
6. Commit e push.
7. Abrir GitHub Pages e auditar visualmente.

## Proibido
- Subir mudança relevante sem atualizar o site.
- Atualizar site sem atualizar memória Markdown quando a mudança for conceitual.
